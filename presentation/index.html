ItemVault

Personal inventory manager — design & architecture highlights

June 5, 2026

Overview

ItemVault helps users record, organize, search, and update belongings in a clean dashboard. Quickly see owned items, total value, location, and status (owned, sold, lost, borrowed).

Problem & Solution

  • Problem: Users lose track of personal items and purchase history.
  • Solution: A simple dashboard with search, filters, and item detail pages.

Design Goals

Simple Organization

Search, categories, and stats on one screen.

Secure Data

Firebase Auth + Firestore user-scoped data.

Consistent Experience

Cards, tokens, and rounded controls across screens.

Visual System

#3730a3

System sans fonts, rounded cards, friendly color tokens.

User Flow

Home → Login/Register → Dashboard → Add/View Item → Edit/Delete

Key Screens

Home

Intro, CTA to dashboard or add item.

Dashboard

Search, filters, stats, item cards.

Item Detail

Image, metadata, edit & delete actions.

Components

  • ProductForm — add/edit validation
  • ProductCard — scannable item card
  • SearchBar & FilterDropdown — reusable controls

Routing & Protection

Public: /, /login, /register. Protected: /dashboard, /add-item, /items/:id, /items/:id/edit

Data & Authentication

  • Firebase Auth (email/password + Google)
  • Firestore: user-scoped items, real-time snapshots

Accessibility & Responsive

Visible labels, alt text for images, readable contrast, responsive grid layouts.
